极简用法

import { AgentSwarmOrchestrator, RequirementAgent, ArchitectureAgent,
         CodeGenAgent, ReviewAgent, TestAgent } from 'agent-swarm';

const orch = new AgentSwarmOrchestrator();
orch.registerAgent(new RequirementAgent(config));
// ... 注册其余 4 个 Agent ...

const result = await orch.executeTask(
  '写一个 Node.js 命令行工具，用来统计目录下所有文件的行数'
);
// result.artifacts → [需求分析, 架构设计, 代码, 审查报告, 测试用例]
// 产物自动写入 workspace/ 目录

核心设计

链式上下文传递

前一个 Agent 的输出是后一个的输入——需求喂给架构，架构喂给编码，链条不断。

流式推理可见

每个 Agent 调用 LLM 时，推理内容实时输出到终端，不是黑盒。

两级容错

API 挂了？每个 Agent 自带 fallback 实现。断网也能跑出完整产物。

类型安全第一

全项目 TypeScript strict mode。11 个源文件，零编译错误，零 LSP 诊断。
